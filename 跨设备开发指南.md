# 📱 跨设备开发和 GitHub 同步指南

本指南详细说明如何在不同设备间同步 Azure Performance Monitor 项目代码，以及如何使用 GitHub 进行版本控制。

---

## 📋 基本信息

- 项目名称：Azure Performance Monitor
- 开发平台：Trae AI
- GitHub 仓库地址：**https://github.com/fsyinghua/AzurePerformanceMonitor.git**
- 最后更新时间：202X年X月X日

---

## 🔄 当前设备保存项目

代码已成功上传到 GitHub！如需在当前设备进行变更，按以下步骤操作：

```bash
cd c:\Codes\AzurePerformanceMonitor

# 查看当前状态
git status

# 添加变更到暂存区
git add .

# 提交变更
git commit -m "你的变更描述"

# 推送到远程仓库
git push origin master
```

---

## 📥 新设备克隆项目

在新设备上获取并开始开发项目：

### 1. 克隆 GitHub 仓库
```bash
git clone https://github.com/fsyinghua/AzurePerformanceMonitor.git
cd AzurePerformanceMonitor
```

### 2. 安装依赖和启动服务

#### 后端服务
```bash
cd backend
npm install
npm run dev
```
- 服务运行于: http://localhost:3000

#### 前端应用
```bash
cd frontend
npm install
npm run dev
```
- 服务运行于: http://localhost:5173

---

## 🚀 新设备上恢复 Trae AI 上下文

在新设备的 Trae AI 对话中恢复开发上下文：

1. 打开项目根目录中的 `TRAE_AI_CONTEXT.md` 文件
2. 复制文件中的所有内容
3. 在新的 Trae AI 对话中粘贴并提交
4. Trae AI 将恢复到你之前的开发状态

---

## 🔄 更换设备后的同步

当从另一台电脑切换回原电脑时：

```bash
cd c:\Codes\AzurePerformanceMonitor

# 更新本地仓库到最新状态
git pull origin master

# 检查状态和日志
git status
git log
```

---

## 📌 关键提醒

- 确保每次完成工作后及时推送代码到 GitHub
- 新设备上需要重新安装所有 npm 依赖
- 如果遇到子模块问题，可以运行 `git submodule update --init`
- 请始终保持 `.gitignore` 文件的正确配置
- 定期检查项目状态和提交历史
